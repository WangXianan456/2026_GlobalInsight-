# 腾讯云 TKE 最小化部署操作指南

本指南将指导你如何以最低成本在腾讯云容器服务 (TKE) 上部署 GlobalInsight 项目。

## 一、 购买 TKE 集群配置 (推荐)

为了在最小成本下跑通全流程分布式部署，我们建议购买以下配置：

1.  **登录腾讯云控制台** -> **容器服务 (TKE)**。
2.  点击 **新建集群** -> 选择 **标准集群** (Standard)。
3.  **基本信息**:
    *   **地域**: 选择离你最近的区域（如广州、上海）。
    *   **网络**: 建议选择 **Global Router** 网络插件，VPC 如果没有现有可用的，新建一个默认的即可。
4.  **Master 节点**:
    *   选择 **托管集群** (Master 节点由腾讯云管理，通常更省心且无须为 Master 节点付费)。
5.  **Node 节点 (关键成本)**:
    *   **机型**: 选择 **标准型 S5** (Intel) 或 **标准型 SA2** (AMD，通常更便宜)。
    *   **配置**: **4核 8GB**。
    *   **数量**: **2 台**。
        *   *解释*: 总共 16GB 内存。Kafka/ES/Flink 等组件优化后约占用 10-12GB，系统预留 2-4GB，刚好够用。
    *   **系统盘**: 默认 50GB 高性能云硬盘即可。
    *   **数据盘**: **勾选**，建议设置 50GB SSD (用于 Kafka/ES 存储数据)。如果只是测试，可以不挂数据盘，直接用系统盘（需修改 StorageClass 或手动配置，建议挂载数据盘省事）。
    *   **计费模式**: 这里的两台机器建议 **按量计费** (测试完可随时释放) 或 **包年包月** (如果长期运行)。
    *   **公网带宽**: 为了操作方便，建议给节点购买小额带宽 (如 1Mbps) 或 **按使用流量计费** (推荐，测试期间流量费很低)，以便拉取 Docker 镜像和本地连接。

## 二、 配置 Kubectl 远程连接

1.  集群创建完成后（需等待约 10 分钟），进入 **集群详情** 页面。
2.  点击 **基本信息** -> **开启外网访问** (如果需要本地操作) 或 **内网访问** (如果你在云服务器上操作)。
3.  下载 **Kubeconfig** 文件。
4.  将内容复制到你本地机器的 `~/.kube/config` (Linux/Mac) 或 `C:\Users\你的用户名\.kube\config` (Windows)。
5.  测试连接:
    ```powershell
    kubectl get nodes
    ```
    应该能看到 2 个 Ready 状态的节点。

## 三、 本地代码配置修改 (已自动完成)

为了适应上述的小规格集群，我已经帮你修改了 `k8s-deploy` 目录下的所有配置文件，将内存限制大幅下调：

*   **Kafka**: 堆内存 512M，容器限制 1.5G。
*   **Elasticsearch**: 堆内存 512M，容器限制 1.5G。
*   **Flink JobManager**: 1G 内存。
*   **Flink TaskManager**: 每个 1.5G 内存 (共 2 个)。
*   **Zookeeper/Redis/Prometheus**: 限制在 256M-512M 之间。

**无须手动修改，直接进行下一步部署即可。**

