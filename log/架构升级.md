## 计划：GlobalInsight 数据平台架构升级

本计划旨在将现有的 **GlobalInsight** Flink 项目升级为一个全栈数据平台。通过集成**离线数仓 (Hive)** 和**实时 OLAP (ClickHouse)**，并增强模拟数据生成功能，以支持更复杂的业务测试。

### 核心步骤

#### 1. 基础设施扩展 (Docker)

- **修改配置**：更新 `GlobalInsight-Env/docker-compose.yml`，添加 `clickhouse-server` 服务（端口 8123, 9000）。

- **离线组件**：在同一个 Compose 文件中添加轻量级 Hadoop/Hive 栈（包括 Namenode、Datanode、Hive Metastore），构建离线数仓。

- **网络打通**：确保 Flink 容器能够通过默认 bridge 网络访问这些新服务。


#### 2. 模拟数据升级 (会话与画像)

- **代码更新**：修改 `Code/src/main/java/MockDataGenerator.java`。

- **字段扩展**：在 `GlobalBehavior` 类中新增 `age` (int)、`level` (String) 和 `sessionId` (String) 字段。

- **逻辑重构**：重构 `generateData` 方法以模拟有状态的“会话 (Session)”：创建一个 `UserSession` 对象来跟踪用户的完整路径（搜索 -> 点击 -> 加购 -> 购买）。


#### 3. 依赖管理

- **更新 `Code/pom.xml`**，添加以下必要依赖：

    - **ClickHouse 相关**：`flink-connector-jdbc` 和 `clickhouse-jdbc`。

    - **Hive 相关**：`flink-connector-hive`、`hive-exec` (建议 3.1.x) 以及 `flink-shaded-hadoop-3-uber`。


#### 4. 实时 OLAP 应用 (ClickHouse)

- **新建应用**：创建 `Code/src/main/java/ClickHouseSinkApp.java`。

- **任务实现**：实现一个 Flink 流处理任务，从 Kafka (`global_behavior_log`) 读取数据。

- **数据写入**：将 JSON 数据映射为 POJO，并使用 `JdbcSink` 或 `ClickHouseSink` 写入 ClickHouse。


#### 5. 离线数仓应用 (Hive)

- **新建应用**：创建 `Code/src/main/java/HiveSinkApp.java`。

- **元数据配置**：在 Flink 中配置 `HiveCatalog` 以注册 Hive Metastore。

- **分区写入**：实现 `Streaming File Sink`（需启用 Checkpoint），将数据写入 Hive 分区表（例如按天 `dt` 和小时 `hr` 分区）。
